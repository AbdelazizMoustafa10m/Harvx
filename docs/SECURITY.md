# Security

## Supply Chain Security

Harvx releases are signed and include a Software Bill of Materials (SBOM) to enable verification of artifact authenticity and dependency auditing.

### Artifact Signing

All release checksums are signed using [Cosign](https://docs.sigstore.dev/) keyless signing via Sigstore. The signing identity is the GitHub Actions workflow that produces the release.

### Verifying a Release

#### 1. Download release artifacts

Download the binary archive, checksum file, signature, and certificate from the [GitHub Releases](https://github.com/harvx/harvx/releases) page:

```bash
# Example for Linux amd64
curl -LO https://github.com/harvx/harvx/releases/download/vX.Y.Z/harvx_X.Y.Z_linux_amd64.tar.gz
curl -LO https://github.com/harvx/harvx/releases/download/vX.Y.Z/harvx_checksums.txt
curl -LO https://github.com/harvx/harvx/releases/download/vX.Y.Z/harvx_checksums.txt.sig
curl -LO https://github.com/harvx/harvx/releases/download/vX.Y.Z/harvx_checksums.txt.pem
```

#### 2. Verify the checksum signature with Cosign

```bash
cosign verify-blob \
  --signature harvx_checksums.txt.sig \
  --certificate harvx_checksums.txt.pem \
  --certificate-identity-regexp "https://github.com/harvx/harvx" \
  --certificate-oidc-issuer "https://token.actions.githubusercontent.com" \
  harvx_checksums.txt
```

A successful verification prints `Verified OK`.

#### 3. Verify the archive checksum

```bash
sha256sum --check --ignore-missing harvx_checksums.txt
```

On macOS (using `shasum`):

```bash
shasum -a 256 --check --ignore-missing harvx_checksums.txt
```

### SBOM (Software Bill of Materials)

Each release archive has an accompanying SPDX JSON SBOM generated by [Syft](https://github.com/anchore/syft). SBOM files are named `harvx_X.Y.Z_<os>_<arch>.tar.gz.spdx.json` and are attached as release artifacts.

You can inspect the SBOM with:

```bash
# View with jq
cat harvx_X.Y.Z_linux_amd64.tar.gz.spdx.json | jq .

# Validate SPDX format
syft validate harvx_X.Y.Z_linux_amd64.tar.gz.spdx.json
```

## Reporting Vulnerabilities

If you discover a security vulnerability, please report it responsibly by emailing [security@harvx.dev](mailto:security@harvx.dev). Do not open a public issue for security vulnerabilities.
